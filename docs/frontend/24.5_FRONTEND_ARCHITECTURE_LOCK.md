## 3. Decisions

### Styling: Tailwind CSS

**Locked:** Tailwind CSS v4 with PostCSS nesting

**Rules:**
- Utility-first for all standard UI
- Custom colors in `tailwind.config.js`
- No arbitrary values unless one-off
- Component extraction for repeated patterns
- Scoped CSS only for: garment tag builder, canvas components

**Rejected:**
- CSS Modules
- Styled-components
- Emotion
- Bootstrap

### Forms: React Hook Form + Zod

**Locked:** RHF + Zod for all forms with ≥3 fields

**Rules:**
- Zod schema defined in `src/schemas/`
- `@hookform/resolvers` bridge
- `defaultValues` always provided
- Error messages via `zod` `.message()`
- Async validation for unique fields (debounced)

**Simple forms** (<3 fields) may use controlled inputs without RHF.

**Rejected:**
- Formik
- React Final Form
- Plain controlled inputs for complex forms

### State Management

**Locked:** React Context for global state

**Rules:**
- AuthContext for auth state
- LanguageContext for i18n
- OrderContext for order draft (customer-web)
- No Redux unless scaling demands it

**Rejected (for now):**
- Redux Toolkit
- Zustand
- Jotai
- Recoil

### Data Fetching

**Locked:** Axios + custom hooks

**Rules:**
- API functions in `src/api/` per domain
- React Query / SWR considered for future (not MVP)
- Loading states managed per-component
- Error boundaries for API failures

### Component Patterns

**Locked:** Composition over configuration

**Rules:**
- PageTemplate for list pages
- StepModal for multi-step flows
- TableCard for table containers
- Props over render props where possible
- Children composition for flexible layouts

## 4. File Organization

```
src/
  api/           # API service functions
  components/    # Shared components
  constants/     # App constants
  context/       # React contexts
  data/          # Mock data, seed data
  hooks/         # Custom hooks
  i18n/          # Translations
  pages/         # Route components
  schemas/       # Zod schemas
  styles/        # Global styles, Tailwind config
  types/         # TypeScript types
  utils/         # Utility functions
```

---

## 5. Pass Criteria

Before any frontend code is approved:

- [ ] Tailwind classes only (no inline styles)
- [ ] RHF + Zod for forms ≥3 fields
- [ ] TypeScript strict mode passing
- [ ] No `any` types without justification
- [ ] Components follow design system
- [ ] Responsive on tablet and desktop
- [ ] Accessible (keyboard nav, labels)

---

## 6. Migration Rules

When converting legacy JanLunMS pages:

1. Strip all inline styles
2. Replace Bootstrap classes with Tailwind
3. Extract forms to RHF + Zod
4. Use PageTemplate for list pages
5. Replace jQuery with React state
6. Remove transport terminology

---

## 7. Enforcement

This architecture lock is binding.

Any deviation requires:
1. Written justification
2. Review by tech lead
3. Update to this document
